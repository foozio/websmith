# Contributing to Websmith Kit

Thank you for your interest in contributing to Websmith Kit! We welcome contributions from the community.

## Code of Conduct

This project follows a code of conduct to ensure a welcoming environment for all contributors.

## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker to report bugs or request features
- Provide detailed information including steps to reproduce
- Include your environment details (OS, Node version, etc.)

### Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/websmith-kit.git`
3. Install dependencies: `npm install`
4. Create a feature branch: `git checkout -b feature/your-feature`
5. Make your changes
6. Run tests: `npm run test`
7. Run linting: `npm run lint`
8. Commit your changes: `git commit -m "Add your feature"`
9. Push to your fork: `git push origin feature/your-feature`
10. Create a Pull Request

### Component Development

When adding new components:

1. Follow the existing component structure
2. Include TypeScript types
3. Add proper accessibility attributes
4. Include unit tests
5. Update documentation
6. Add to the playground

### Design Token Updates

When modifying design tokens:

1. Update the token definitions
2. Ensure all export formats work
3. Update any dependent components
4. Test theme compatibility

### Documentation

- Keep documentation up to date
- Use clear, concise language
- Include code examples
- Test documentation builds

## Commit Guidelines

- Use clear, descriptive commit messages
- Follow conventional commit format when possible
- Reference issue numbers in commits

## Testing

- Write unit tests for new features
- Ensure all existing tests pass
- Test components across different browsers
- Test accessibility features

## Questions?

If you have questions about contributing, please open an issue or contact the maintainers.
