## üìù Documentation Guidelines

### File Organization
- Root-level docs: Project-wide information
- `/docs`: Consolidated guides and references
- `/packages/*/`: Package-specific documentation

### Naming Conventions
- Use UPPERCASE for root-level docs (README.md, CONTRIBUTING.md)
- Use PascalCase for consolidated docs (Development.md, Architecture.md)
- Use descriptive names (not abbreviations)

### Content Guidelines
- Keep docs DRY (Don't Repeat Yourself)
- Link to other docs instead of duplicating
- Update related docs when making changes
- Include examples and code snippets
- Keep language clear and concise

## üîÑ Maintenance

### Regular Updates
- Review docs quarterly
- Update after major features
- Keep examples current
- Fix broken links

### Deprecation Process
1. Mark as deprecated in doc
2. Add migration guide
3. Update references
4. Remove after 2 major versions
